# База знаний отдела Engineering

## Правило 1: Code Review
Все pull request должны пройти review минимум от 2 разработчиков перед мержем в main.

## Правило 2: Время деплоя
Деплой в production разрешён только с 10:00 до 16:00 по московскому времени, кроме экстренных случаев.

## Правило 3: Naming Convention
Переменные должны быть в camelCase, константы в UPPER_SNAKE_CASE, классы в PascalCase.

## FAQ: Как получить доступ к staging?
Для доступа к staging окружению нужно создать заявку в Jira с типом "Access Request" и указать причину.

## FAQ: Какой максимальный размер PR?
Рекомендуемый размер pull request — не более 400 строк изменённого кода. Большие PR нужно разбивать на части.
