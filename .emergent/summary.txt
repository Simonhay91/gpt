**PRODUCT REQUIREMENTS:**
*   **Core App:** Multi-user application with project-based data isolation using React, FastAPI, and MongoDB.
*   **User Management:** Admin-centric model where an admin creates users, can view detailed usage statistics (tokens, messages, storage), and manage user-specific settings like GPT models and prompts.
*   **GPT Configuration:** A global admin-configurable GPT system prompt and model, with overrides available on a per-user basis, also managed by the admin.

    *   Project-based chats with history. Quick Chats for non-project work.
    *   Ability to rename chats and move them between projects.
    *   **Project Sharing & Roles:** Owners can share projects with other users, assigning roles (e.g., Viewer, Editor, Manager) to control permissions.
    *   Fine-grained Chat Visibility: Owners can control which specific chats are visible to each shared user within a project.
    *   AI responses must have a Copy button and render clickable URLs.
    *   Display the sender's name on messages in shared chats.
*   **Source Management:**
    *   Users can attach various file types (PDF, DOCX, TXT, MD, PPTX, XLSX, PNG, JPEG, CSV). The system must handle encrypted PDFs and large files.
    *   **OCR:** Text must be automatically extracted from uploaded images (PNG/JPEG).
    *   All sources within a project are always active for any chat in that project.
    *   **Source Search:** Implement a token-free search functionality to find content within a project's sources without calling the LLM.
    *   **Global Sources:** An admin-managed central knowledge base that is automatically available to all users. Permissions can be granted to non-admin users to manage these sources.
    *   **Source Usage Analytics:** Admins can view statistics on how frequently sources are used in GPT responses.
*   **Efficiency & Cost:**
    *   **Semantic Caching:** Implement an intelligent caching layer using embeddings to find semantically similar questions and return cached answers to reduce token usage. The cache must be securely isolated by project, user permissions, and prompt configurations.
*   **UX/UI:**
    *   The app should have light and dark modes.
    *   Fix navigation so the back button in a chat returns to the project's chat list, not the main dashboard.
