**Key Technical Concepts**
- **In-Chat Tool Use:** A sophisticated pattern where user intent is detected from natural language in a chat message, triggering a specific backend tool-use flow (LLM-driven data transformation) and returning a structured result (downloadable file) into the UI.
- **System Prompt Engineering:** Dynamically building and modifying system prompts sent to the LLM to guide its behavior for specific tasks like web search or data transformation, including adding negative constraints (e.g., Do not generate XML).
- **Fixing Optimistic UI Bugs:** Resolved a critical Message not found bug by refactoring the message-sending flow. The backend now persists the user message and returns both the user's and the AI's messages with their real database IDs, which the frontend then uses to update its state, eliminating reliance on temporary frontend IDs.
